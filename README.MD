## Архитектура системы и технические решения

Для обеспечения масштабируемости и чистоты кода проект разделен на специализированные сервисы:

| Сервис | Порт | Описание |
| :--- | :--- | :--- |
| **Auth Service** | `8001` | Авторизация и генерация JWT токенов. |
| **Core Service** | `8002` | Основная бизнес-логика и сбор статистики. |
| **Landings Service** | `8003` | Прием лидов с лендингов и взаимодействие с брокером. |
| **Redis Broker** | `—` | Фоновый воркер, переносящий данные из очереди в БД. |

### 💡 Особенности реализации
* **Выделенный брокер:** Брокер Redis вынесен в отдельный скрипт. Логически он принимает данные от `landings` и сохраняет их в БД, поэтому он работает как независимый фоновый воркер.
* **Автоматическая инициализация:** Так как CRUD для офферов и аффилиатов не требовался по ТЗ, в сервисе `landings` они создаются по одному экземпляру автоматически для удобства тестирования.
* **Изоляция авторизации:** Сервис `auth` работает независимо, имитируя реальную среду Single Sign-On (SSO).

## 🛠 Установка и запуск

Все команды должны выполняться из корня проекта (`test_task_leads`).

### 1. Конфигурация окружения
Создайте файл `.env` в корневой папке и заполните его данными, следуя схеме из `app/config.py`:
```env
# Пример (настройте под свои параметры)
DB_HOST=127.0.0.1
DB_PORT=5435
DB_USER=postgres
DB_PASS=12345678
DB_NAME=leads_db

REDIS_HOST=127.0.0.1
REDIS_PORT=6379

JWT_SECRET_KEY=secret_key
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
```
### 2. Запуск инфраструктуры
Запустите базу данных и Redis через Docker Compose:
```Bash
docker compose up --build --detach
```

### 3. Миграции базы данных
Примените актуальную схему базы данных с помощью Alembic:

```Bash
alembic upgrade head
```
## 🚀 Запуск сервисов
#### Для корректной работы запустите каждый скрипт в отдельном окне терминала:

#### 1. Сервис авторизации:

```Bash
uvicorn "app.auth_service.main:app" --reload --port 8001
```

#### 2. Сервис Core:

```Bash
uvicorn "app.core_service.main:app" --reload --port 8002
```

#### 3. Сервис Landings:

```Bash
uvicorn "app.landings_service.main:app" --reload --port 8003
```
#### 4. Брокер сообщений (обязательно):

```Bash
python3 app/broker.py
```

## 🔍 Тестирование и документация
#### После запуска всех сервисов интерактивная документация Swagger доступна по следующим адресам:

#### Auth: http://localhost:8001/docs

#### Core: http://localhost:8002/docs

#### Landings: http://localhost:8003/docs

## ⚠️ Важные примечания
#### 1. Важно: Все скрипты должны запускаться строго из корня проекта для корректного импорта модулей.

#### 2. Очередь сообщений: Лиды, созданные через landings_service, не появятся в базе данных и статистике до тех пор, пока не запущен app/broker.py.

#### Язык: Python 3.12+

#### Фреймворк: FastAPI
